[INTRO — help screen visible, ~36s]
email-cli is a terminal email client written in C.
It has exactly two external runtime dependencies: OpenSSL and libcurl.
Both are decades-old, system-level libraries
shipped and verified by your Linux distribution.
No package manager, no runtime framework, no supply-chain risk.
The suite consists of four independent binaries with distinct privilege levels:
an interactive TUI, a read-write batch CLI,
a strictly read-only variant designed for AI agents,
and a background sync daemon.
All four share the same configuration and local message store.

[SHOW-ACCOUNTS — ~8s]
show-accounts lists every configured account.
Each account has its own configuration directory.

[LIST — ~9s]
The list command shows unread messages from the local cache.
A server connection is only needed during sync.

[SHOW — before ~5s, after ~10s]
The listing shows each message's UID and subject.
The show command prints the full message content by UID —
headers, body, no HTML —
directly to stdout, ready for any script or AI tool to process.

[LIST --all + GREP — before ~5s, after ~7s]
By default, list shows only unread messages.
The --all flag includes everything,
and the output can be piped directly into grep or any other tool.

[FOLDERS — ~9s]
folders --tree draws the full folder hierarchy.
For IMAP accounts it shows mailbox folders;
for Gmail accounts it shows labels.

[LIST --folder — before ~5s, after ~7s]
list defaults to the INBOX folder.
The --folder flag opens any other folder directly.

[ATTACHMENTS — before ~5s, after ~7s]
Messages can contain attachments.
The attachments command lists each attachment's filename and size.

[SAVE-ATTACHMENT — before ~4s, action ~4s, after ~7s]
The file does not exist on disk yet.
save-attachment downloads a single attachment by UID and filename
and writes it to the specified directory.
After saving, the file is present on disk.

[SEND — before ~5s, action ~6s, after ~8s]
Before sending, the Sent folder shows its current contents.
The send command delivers the message over SMTP
and automatically saves a copy to the Sent folder.
After sending, the new message appears in Sent.

[MARK-READ — before ~6s, action ~3s, after ~7s]
The message is currently unread and appears in the list.
mark-read sets the Seen flag on the server.
After marking, the message is gone from the unread list.

[MARK-UNREAD — action ~3s, after ~7s]
mark-unread clears the Seen flag.
The message reappears in the unread list.

[MARK-STARRED — before ~6s, action ~3s, after ~7s]
The message currently has no star.
mark-starred sets the Flagged flag on the server.
After starring, the message appears among flagged messages.

[REMOVE-STARRED — action ~3s, after ~7s]
remove-starred clears the flag.
The message returns to its unflagged state.

[CONFIG SHOW — ~9s]
config show prints the current account configuration.
Passwords and tokens are shown masked with asterisks.

[SYNC — ~11s]
email-sync fetches new messages from the server
and updates the local message store.
Run from cron, it keeps the store current
so every other tool can list instantly without a live server connection.

[OUTRO — ~12s]
email-cli — C, POSIX, OpenSSL, libcurl.
Source code, documentation, and issue tracker: github.com/csjpeter/email-cli.
